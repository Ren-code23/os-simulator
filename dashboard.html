Deadlock / Resource Allocation

Banker's Algorithm

What it does: Prevents deadlock by checking if resource allocation keeps the system in a safe state.

How it works:

  1. Calculate Need matrix (Max - Allocation)
  2. Find a process that can finish with available resources
  3. If found, assume it finishes and releases resources
  4. Repeat until all processes finish (SAFE) or none can (UNSAFE)

When to use: Resource allocation in operating systems to avoid deadlock.

Page Replacement Algorithms

Page Replacement Algorithms

What they do: Decide which page to remove from memory when a new page is needed.

Algorithms:

  • FIFO: Replace the oldest page (first in, first out)
  • LRU: Replace the least recently used page
  • Optimal: Replace the page that won't be used for longest time (theoretical best)

Goal: Minimize page faults (when requested page isn't in memory).

CPU Scheduling Algorithms

CPU Scheduling Algorithms

What they do: Decide which process gets CPU time and when.

Algorithms:

  • FCFS: First-Come, First-Served - processes run in order of arrival
  • SJF: Shortest Job First - shortest processes run first
  • Priority: Higher priority processes run first
  • Round Robin: Each process gets a time quantum in circular order

Metrics: Waiting time (time waiting in queue), Turnaround time (total time from arrival to completion)
